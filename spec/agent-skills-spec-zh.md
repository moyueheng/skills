# Agent 技能规范

技能是一个包含指令、脚本和资源的文件夹，代理可以动态发现和加载这些内容，以便在特定任务中表现更出色。要让文件夹被识别为技能，必须包含一个 `SKILL.md` 文件。

# 技能文件夹布局

一个最小化的技能文件夹结构如下：

```
my-skill/
  - SKILL.md
```

更复杂的技能可以根据需要添加额外的目录和文件。

# SKILL.md 文件

技能的"入口点"是 `SKILL.md` 文件。这是唯一必须存在的文件。该文件必须以 YAML 前置内容开头，后跟常规的 Markdown 内容。

## YAML 前置内容

YAML 前置内容有 2 个必需属性：

- `name`
    - 技能的名称，使用连字符分隔格式
    - 仅限于小写 Unicode 字母数字字符和连字符
    - 必须与包含 SKILL.md 的目录名匹配
- `description`
    - 描述技能的功能以及 Claude 应该在何时使用它

还有 3 个可选属性：

- `license`
    - 应用于技能的许可证
    - 建议保持简短（许可证名称或许可证文件名）
- `allowed-tools`
    - 预先批准运行的工具列表
    - 目前仅在 Claude Code 中支持
- `metadata`
    - 从字符串键到字符串值的映射
    - 客户端可以用它来存储 Agent Skills Spec 未定义的其他属性
    - 建议让键名足够独特以避免意外的冲突

## Markdown 正文

Markdown 正文没有限制，可以自由编写。
